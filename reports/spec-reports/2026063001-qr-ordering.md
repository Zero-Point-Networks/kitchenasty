# Spec Report — QR / Dine-in Ordering

Date: 30 June 2026 | Session: interactive

## What Was Delivered

Diners can now scan a per-table QR code to order without staff taking the order. Scanning opens the storefront menu already bound to the table; the diner builds an order and checks out as a **dine-in** order, choosing **Pay at Counter** (placed unpaid, settled by staff) or any enabled online method. A motel guest can order from their room and collect at the restaurant. Front-of-house staff are freed from order-taking.

- **Storefront**: a scanned code (`/t/<token>`) resolves the table and opens the menu; the checkout shows a "Dine-in — {table}" banner, hides the delivery-address step, allows anonymous ordering, and offers "Pay at Counter". Invalid/expired codes show a recoverable "Table not found" page.
- **Admin**: the Tables page (Locations → a location → Tables) gains a **Generate QR / Regenerate QR** action per table that opens a printable QR modal; regenerating warns that printed copies are invalidated.
- **Operator control**: dine-in ordering is gated by a `dineInEnabled` order setting — it is off unless enabled (and is enabled by the demo seed).
- **Generic & upstreamable**: nothing motel-specific is hardcoded; dine-in is a first-class order type alongside delivery/pickup.

## How to Verify

Preconditions: a Postgres database, `prisma migrate deploy` (applies the new migration), `prisma db seed` (creates Table 1 with QR token `dev-table-1-qr` and enables dine-in), and the server + storefront + admin running.

1. **Admin QR**: open Admin → Locations → (a location) → Tables. Click **Generate QR** on a table. **Expected**: a modal with a QR image + the `/t/<token>` URL and a Print button; the row's action becomes **Regenerate QR**.
2. **Scan flow**: visit `http://localhost:5174/t/dev-table-1-qr`. **Expected**: redirect to `/menu`.
3. **Dine-in checkout**: add an item, go to `/checkout`. **Expected**: a "Dine-in — Table 1" banner, no address step, a "Pay at Counter" payment option, optional contact fields.
4. **Place order**: with "Pay at Counter" selected, place the order. **Expected**: order is created (`orderType: DINE_IN`, `tableId` set) and appears in the admin order list; lands on the confirmation page.
5. **Invalid token**: visit `/t/bogus`. **Expected**: "Table not found" with a Browse Menu link.
6. **Gate**: set `orderSettings.dineInEnabled` false; a dine-in order POST returns 400 "Dine-in ordering is not currently available".
